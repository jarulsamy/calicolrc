1. The Calico Project

The Calico Project was designed to allow introductory computer science students to program robots in Python. It is the product of a seed grant from Microsoft Research, and NSF DUE grant #0920539 supporting the Institute for Personal Robots in Education run by Georgia Tech with Bryn Mawr College.

1.1 Why is Calico interesting?

  • Supports a number of different languages, including Java, Python, Ruby, F#, Scheme, Basic, Logo, LC3 (assembly language), a block-based language called Jigsaw, and others
    • Spans beginning to advanced computing languages
    • Languages all operate in the same space---can share data and functions
  • Contains a number of different modules for exploring different contexts in computer science, including:
    • robots and devices (Myro)
    • mediacomp-style projects (Graphics, sound, animation, ...)
    • art (Processing)
    • robotics, ai, machine learning, etc
    • Spans beginning to advanced topics
    • Libraries are written once, and appear native to each language (eg, as blocks in Jigsaw, as Scheme functions in Scheme, as Python in Python)
      • Once a teacher learns a library API, it is constant across all uses - contexts, languages, courses
      • Consistent interface and library API facilitate transitions and transfer in student learning
      • A single executable and set of libraries runs identically on all operating systems -- easy to maintain

Consequences:

  • Independent of a particular language's low-level library ecosystem (eg, C-based Python libraries, or Scheme modules)
  • Calico libraries are language neutral

CS Educators should not be tied to a particular language, but have the freedom to move to a better one if when it appears (even if it happens to be, say, "Python with braces" or "Pascal")

2. Literate Computing

Recently, my colleagues and I have been interested in a notion that we call Reading, Writing, and Computing --- an integration of computation into the Liberal Arts philosophy. Fernando Perez's Literate Computing captures that essence. We believe that:

  1. Students should be able to Read STEM-based materials
  2. Students should be able to Write STEM-based materials
  3. Computation should be a core part of STEM-based materials

In a nutshell, we were exploring the ideas that students should be better able to understand computer science writings---and producing their own should help in that endeavour.

On January 16, 2014, we began investing IPython as the core of these "STEM-based materials."

[1] - Doug Blank, Jennie Kay, Jim Marshall, and Keith O'Hara.

3. ICalico

ICalico is a new project that makes the old Calico GUI available through the IPython interface.

"IPython" is actually 5 different things:

  1. A language "kernel" for interpreting expressions
  2. Three "frontends" for talking to a kernel (console, qtconsole, and notebook)
  3. The notebook is an API and implementation for a set of Widgets for language + web interaction
  4. A JSON socket-based message-passing API and implementation for communication
  5. A program that manages all of the above, ipython

Fernando is the Tim Berners-Lee of Notebook Computing.

(Ironically, none of these need to have anything to do with Python)

Example

The net input is the sum of the activation times weight from the incoming units, plus a bias value.

$net_i = \sum\limits_{j=1}^n w_{ij} x_j + b_i$

3.3 Future

  1. Work with the IPython team to develop better support for "third-party kernels" and extended functionality (can we have stepper support in IPython?)
  2. Develop a notebook server designed for the educational context
    1. Login, cloud-based
    2. Develop
    3. Computational notebooks could be created that provide the labels for a group of steps that assist students in "filling in the blanks" in a task along the lines of Catrambones subgoal learning model (1998).
    4. Computational notebooks are a wonderful medium for worked examples (Sweller, 2006) in which students can be stepped through the solution to a problem, providing them with code snippets as they progress.
    5. By combining explanations with images and executable code, computational notebooks could significantly reduce the cognitive load (Sweller, 1998) that novices face in many STEM fields.
    6. Our plan to ask students in our classes to produce their own computational notebooks that can be shared with their peers will provide numerous opportunities for peer instruction (Mazur, 1997).

4. Summary

  • If you want to get involved now, have an educational influence over development; or wait and use later, after it hads been developed and refined
  • IPython interface is great for teachers now, even if your students aren't using it yet
    • Students don't notice that you are using a different interface
    • Makes in-class demos easy to do and engaging
    • Side-effect: record of what you did in class
    • Great use in the flipped classroom setting
  • ICalico is ready for your envolvement
